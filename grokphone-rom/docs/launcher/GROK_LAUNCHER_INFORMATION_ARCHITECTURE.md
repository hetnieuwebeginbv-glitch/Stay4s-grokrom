# GROK_LAUNCHER_INFORMATION_ARCHITECTURE.md

## Overview

The Information Architecture of the Grok Launcher is designed to serve the Parallel Grok Brain and Daily Guardian while maintaining extreme calm and clarity for the owner.

It follows a strict hierarchy that prevents information overload while enabling deep capability when needed.

## Primary Layers

### Layer 0 – Calm Canvas (Always Present)
- Current time (large, elegant)
- Current location + weather (minimal)
- Single “Today at a Glance” line (optional, user-controlled)
- One primary proactive intelligence surface (if the Guardian + Brain determine high value)

This layer is intentionally information-poor by default. Its purpose is presence and peace, not data consumption.

### Layer 1 – Contextual Intelligence Surface
This is the most important architectural element.

**Card Types (prioritized by the Parallel Brain + Guardian):**

1. **Protective / Guardian Cards** (highest priority)
   - Privacy risks, unusual behavior, security events
   - Covenant-aligned warnings (especially on Genesis devices)

2. **Opportunity Cards**
   - Better networks (Meshmatic peers, Wi-Fi, etc.)
   - Time-saving or energy-saving suggestions
   - Relevant Genesis network activity

3. **Anticipatory Cards**
   - Next likely action based on context and history (via SharedEvolvingContextGraph)
   - Calendar + travel intelligence
   - Health / routine pattern support

4. **Genesis Covenant Cards** (Genesis 001-100 only)
   - Private mesh status
   - Batch-specific insights
   - Ethical or long-term reflection prompts aligned with the original covenant

Each card must carry:
- Clear, human-readable title
- One-sentence explanation
- Reasoning trace button (shows Parallel Brain paths that led to this recommendation)
- Primary action + secondary “Why?” / “Not now” options

### Layer 2 – Grok Engagement Entry Points
- Prominent but calm “Talk to Grok” surface
- Quick actions row (Vault, Meshmatic, Guardian Report, etc.)
- Search / command bar that directly feeds the Parallel Orchestrator

### Layer 3 – Full System Overview (Deeper Navigation)
- Accessible via gesture or dedicated area
- Sections:
  - Guardian Status & Daily Report
  - Grok Vault Overview
  - Meshmatic Network
  - System Intelligence (battery, privacy, performance)
  - Genesis Network (001-100 devices)

## Information Flow Principles

1. **Guardian First**
   The Daily Guardian always gets first right of surface. Protective information outranks convenience.

2. **Brain Reasoning Transparency**
   Every proactive recommendation must be traceable back to the Parallel Grok Brain’s reasoning paths.

3. **Progressive Disclosure**
   - Level 0: Almost nothing
   - Level 1: 1–3 high-signal cards
   - Level 2: Full conversational interface
   - Level 3: Complete system intelligence dashboard

4. **Covenant Alignment (Genesis Devices)**
   On Genesis 001-100 units, an additional invisible layer filters and prioritizes information according to the original covenant values. This layer is not user-configurable in the traditional sense — it is part of the sacred contract.

## Data Sources

- SharedEvolvingContextGraph (long-term personal memory)
- DailyGuardianAgent (real-time protection and opportunity detection)
- ParallelOrchestrator (multi-path reasoning)
- System services (via ISystemActor abstraction)
- Private Genesis Meshmatic network (Genesis devices only)

## Scrolling & Navigation Model

- Vertical scrolling is calm and limited
- Most valuable content lives above the fold
- Deeper content is reached through intentional gestures, not endless feeds
- No infinite scroll. Ever.

## Personalization Boundaries

The launcher learns, but never betrays.

- All learning happens on-device
- The owner can inspect, edit, or delete any part of the ContextGraph at any time
- On Genesis devices, certain categories of data (covenant-related reflections, private mesh activity) receive extra protection layers

---

This architecture ensures the launcher remains a servant of the owner and the covenant, rather than a platform for attention extraction.
